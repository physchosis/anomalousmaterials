  • Author or Got Materials?

    Maybe you have a blog today, a whitepaper, an awesome new code repository, or found some awesome new security or hacking thing that you want to share. How about helping to create some material artifacts or help create some guides on how to do things. The goal of this project is to develop a way of sharing security/hacking informational materials and artifacts in a new way. There is also a simple collaborative app where people can submit new materials, attack and defend guides and articles across the Security Community. Notice I didn't use the word Cyber? Its because a printed 3D Key of a TSA lock...Is not Cyber related.... So whether you have a new project that sends out AM frequencies on a system bus, a new 3D printed project for creating keys, an awesome article, guide, or tutorial, or a new project that defends against attacks the goal is to provide a way of sharing this information, in a somewhat distributed way.

    To create anomalous materials, you have a few options.

    Option 1

    Login and click the create/submit link in the nav. You will then be presented with a simple form, and you can also read more about advanced options.
    Login Now

    Option 2

    Fork/Clone amp-materials repo, add anomalous metadata file, submit pull request.

    Option 3

    For users that want to host their own anomalous metadata file either in their own repo or elsewhere. If you are using github, you can simply create a new repo, or add the metadata file to an existing repo and then submit it using the create/submit form under anomalout/advanced to submit the URL of the anomalous material metadata file. If you are not using github, and you are hosting the metadata file on a server with CORS enabled, then you must provide access for the anomalous materials domain. This is only temporary as there is currently no server backend to process outbound requests. In the future I may add other methods to submit depending on your feedback. For information on CORS see the wikipedia article or determine how to add the appropriate cross origin resource headers for your specific environment.

  • Get to the point! What is the app like???

    Initially the material will be provided visually in a small app in a card format kind of like bookmarketing websites or seen below. The difference is in how the cards linking to the material are constructed. Basically each anomalous material, will use metadata maintained by either the author...or the creator (the person that found it).... At which point each card will have associated options on what you can do with the anomalous material. Maybe you can downloaded it...maybe its encrypted..maybe the material itself is secret code for the user cyber apocalpyse end of the world date...or a puzzle produced by 1o57 for you to solve...For instance if its an artifact that is a video, then it might play the video, if it is an artifact to a code repository, it might show you information regarding the project, possibly snippets of code etc. The metadata may also be linked thus making a material artifact a larger conglomerate.

    Unlike the bookmarketing sites, the idea behind a conglomerate is to group related anomalous materials together. The following is a good history lesson to give you a better idea. Back in the mid-late 90's there was a digital Artist group called ICE.org one of the really cool things they did was make something called Artist Digital Tiles. The digital tiles were a collaboration of artwork combined into one big image. Think of google maps, but that each title consists of someone actually creating the artwork behind the tile. Rumor has it that some of the artist were big in the demoscene, and cracktros back in the day...

  • Create a gh-pages branch

    Head over to GitHub.com and create a new repository, or go to an existing one. In the repository overview, click the branch drop-down on the left-hand side. Type in amp-pages and press enter.

    For Project pages, the gh-pages branch is used to publish your site. That means that you can work with GitHub Pages in the same repository as the project that it's for.

  • Optional: Make gh-pages the default branch

    If you created a new branch for GitHub Pages, go to the repository settings. In the Branches section, change the default branch to the newly created gh-pages branch.

  • Create an index file

    Go back to the repository overview, and use the plus icon next to the repository name to create a new file.
